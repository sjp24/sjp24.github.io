스타트업 데이터 분석가 이력서 #2: 이력서/포트폴리오에 내용

Sep 16, 2022 in Career

스타트업 데이터 분석가 시리즈 1탄 (서류탈락 하는 이력서)에 이어서 그러면 어떤 서류에는 내용을 포함해야 하는지, 데이터 분석가 포트폴리오는 어떻게 구성 해야하는지 얘기해보겠습니다.

1탄에서는 “이렇게 하지마세요”를 설명 했는데요.
그렇다면…어떤 이력서를 제출 해야할까요?



데이터 분석가 이력서/포트폴리오데 대해 자주 받는 질문들 + 평소 생각하는 부분들에 대한 내용입니다.


이력서 형식

이력서를 복잡한 포맷으로 만들거나 디자인을 예쁘게 할 필요는 없어요. 가독성이 중요하기 때문에 심플한게 최고입니다! 템플릿이 필요하다면 깔끔하게 원티드 이력서를 사용하거나 노션 이력서를 만들어보세요.


포트폴리오 (깃헙 포함) 제출 여부

경력이 아예 없는 경우 포트폴리오가 도움이 되긴 합니다. 하지만 데이터 분석가는 포트폴리오에 보여줄 수 있는 내용이 개발자나 디자이너 만큼 명확하지 않기 때문에 제 경험상 포트폴리오 내용이 오히려 실망스러운 경우가 대부분입니다. 코드가 생각보다 너무 심플하거나, A/B 테스트 내용이 너무 단순해서 전혀 임팩트 없어 보이거나, 대시보드 스샷만 엄청 들어가고 설명은 없거나…포트폴리오 정리가 제대로 안되어있고 내용이 별로 없는 경우도 꽤 있습니다. 무조건 보여줄 내용의 양이 많다고 좋은건 아니기 때문에 노력해서 포트폴리오를 의미있게 만들지 않는다면 차라리 이력서에 있는 설명들을 더 잘 쓰고 이력서만 제출하는게 좋을 수 있습니다.


이력서/포트폴리오 내용: 개인 분석 프로젝트

개인 분석 프로젝트는 보통 무료 데이터셋을 사용한 분석 내용이고 실제 어떤 회사에서 회사의 성장이나 서비스 개선을 위해 인사이트를 도출하는게 아니기 때문에 겉할퀴식 EDA 내용과 데이터 시각화로 끝나는 경우가 많아요. 무료 데이터셋으로 한계가 있을 수 있지만, 최대한 인사이트와 인사이트를 바탕한 의견을 알려주세요. “Why”와 “Next Steps”를 찾을 줄 아는 사람인걸 보여주세요. 이런 프레이밍을 잘 하기 위해서는 데이터셋 선택도 중요할거에요. 그냥 포트폴리오를 위한 데이터셋을 찾기 보다 정말 궁금한 분야의 재미있을것 같은 데이터셋을 찾아보세요!


이력서/포트폴리오 내용: 경력에 대한 설명

경력직의 경우, 각 획사에서 진행한 업무 중 주요 업무 내용을 잘 설명하는게 중요합니다. 명확하게 보여줄 수 있는 내용이 만들었던 차트 예시들과 코드 snippet이라서 이런 내용을 많이 포함 하는데요, 대부분 크게 impressive 하지 않습니다. 왜냐하면 사실상 데이터 분석 업무에 중요한 부분은 이런 아웃풋보다 다양한 직무 구성원들과 고민하고 협업한 과정, 코드 짜면서 겪은 어려움들, 분석 내용을 이해 관계자들한테 전달하고 설명했던 방식, 얻었던 인사이트들로 이해관계자들과 논의했던 내용, 분석 바탕으로 개선된 부분들입니다. 이런 내용들을 포트폴리오에 다 담는건 어렵지만, 최대한 프로젝트 배경에 대한 설명과 더불어 분석 과정 (문제 정의, 가설 세우기, 지표 설정, 등등)을 상세하게 설명 해주세요.


SQL? Python? R?

회사마다 그리고 포지션마다 필요한 하드 스킬이 조금 달라서 정답은 없지만 SQL이 가장 기본이고 베이스라고 생각합니다. SQL 고수가 되어야하는건 아니지만 대체적으로 분석을 할때 데이터베이스에 있는 데이터를 사용하기 때문에 SQL을 알아야하고 SQL 지식이 데이터베이스 지식이랑 어느정도 같이 간다고 생각하기 때문에 SQL은 가장 기본으로 알아야한다고 생각해요. 제가 생각하는 중요도는 SQL > Python > R이지만 이게 무조건적인 정답은 아닙니다. 관심 있는 포지션들의 채용 공고 내용을 유심히 보면 어떤 언어를 배워야할지 혹은 어떤 경험을 이력서/포트폴리오에 더 부각 해야할지 알 수 있습니다.


자격증 (SQLD, SQLP, ADsP, 등)

SQLD, 등 이런 류의 자격증은 스타트업에서는 크게 의미 없다고 생각합니다 - 마이너스도 아니지만 플러스도 아닙니다. 일단, 실전 (특히 스타트업에서의) SQL은 시험과 많이 다릅니다. 그렇기 때문에 자격증이 있어도 결국 다른 방법으로 (경력이 없다면 개인 분석 프로젝트와 같은 방법) 분석 역량을 보여줘야해요.


데이터 분석가로 나름(?) 오래 일하면서 이런저런 생각들을 적어봤습니다. 저도 여전히 부족하고, 회사마다 다른 기준들이 있기에 조심스러운 부분도 있는데요, 조금이나마 도움이 됐으면 합니다.
